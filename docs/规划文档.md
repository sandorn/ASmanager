# Agent Skills Manager 规划文档

## 1. 背景

不同 AI Agent 工具对 Skills、MCP Server、提示词和工具配置的支持方式不同，导致同一套能力需要在多个工具中重复维护。Agent Skills Manager 旨在提供一个 VS Code 内的集中管理入口，以 `~/.agent/skills` 作为统一中央仓库，把 Skills 和 MCP 配置统一安装、更新、检测、同步和分发。

## 2. 目标

- 建立一个本机中央 Skills 仓库。
- 支持多个 GitHub Skills 来源。
- 自动检测本机已安装或已配置的 Agent 工具。
- 将中央 Skills 同步到不同工具的目标目录。
- 提供 Skill 质量检查、重复能力分析和冲突提示。
- 后续支持 MCP Server 的配置管理、同步和健康检查。
- 打包为 `.vsix`，方便离线安装和自用分发。

## 3. 非目标

- 第一阶段不做公开市场发布。
- 第一阶段不强依赖某一个商业模型 API。
- 第一阶段不强行修改用户已有 Agent 配置，所有写入动作都需要明确确认。
- 第一阶段不追求覆盖所有 Agent 工具，只优先适配常用目录和可验证配置。

## 4. 核心概念

### 4.1 中央仓库

默认路径：

```text
~/.agent/skills
```

每个 Skill 建议为独立目录，目录内至少包含：

```text
SKILL.md
```

可选包含：

```text
README.md
references/
scripts/
templates/
examples/
```

### 4.2 Skill 来源

初始可支持以下来源：

- `https://github.com/anthropics/skills`
- `https://github.com/openai/skills`
- `https://github.com/MoizIbnYousaf/Ai-Agent-Skills`
- `https://github.com/heilcheng/awesome-agent-skills`

来源信息可保存到扩展全局存储或项目配置文件中。

### 4.3 Agent 适配器

每个 Agent 工具对应一个适配器，用于处理：

- 检测是否安装。
- 检测配置目录。
- 检测 Skills 或 MCP 目录。
- 同步 Skills。
- 回滚或禁用同步。

候选适配目标：

- Claude Code
- Codex
- OpenCode
- Cursor
- Gemini CLI
- Copilot
- Cline
- Roo Code
- Kimi
- Qwen

## 5. 功能规划

### 5.1 Skills 扫描

- 扫描中央仓库目录。
- 识别包含 `SKILL.md` 的目录。
- 提取 Skill 名称、描述、文件数量、大小、更新时间。
- 标记缺失文档或结构异常的 Skill。

### 5.2 来源管理

- 添加来源。
- 删除来源。
- 拉取来源。
- 更新来源。
- 从来源安装指定 Skill。
- 展示来源状态和更新时间。

### 5.3 Agent 检测

- 检测命令是否存在。
- 检测常见配置目录。
- 检测扩展安装状态。
- 展示已支持、已检测、未配置、不可用状态。

### 5.4 同步分发

同步模式：

- 复制模式：兼容性最好。
- 符号链接模式：适合统一更新，但 Windows 可能需要权限。
- Junction 模式：适合 Windows 目录链接。

需要记录同步清单，便于后续更新、禁用和回滚。

### 5.5 质量检测

第一阶段静态检测：

- 是否存在 `SKILL.md`。
- `SKILL.md` 是否包含名称和描述。
- 是否存在空目录。
- 文件数量是否异常。
- 是否包含过大的二进制文件。
- 是否存在重复文件名或疑似重复能力。

第二阶段语义检测：

- Skill 描述相似度。
- 能力重叠提示。
- 风险项提示。
- 推荐合并或拆分建议。

### 5.6 MCP 管理

后续功能：

- 扫描 MCP Server 配置。
- 展示 MCP Server 状态。
- 启用、禁用、更新 MCP Server。
- 将 MCP 配置同步到不同 Agent 工具。
- 健康检查和日志查看。

## 8. 数据存储建议

可优先使用 VS Code 扩展全局存储，后续再支持导出配置文件。

建议保存：

```json
{
  "centralRepository": "~/.agent/skills",
  "sources": [],
  "agents": {},
  "syncRecords": []
}
```

## 9. 阶段计划

### 阶段一：自用 MVP

- 初始化扩展项目。
- 实现中央仓库扫描。
- 实现 Skills Tree View。
- 实现基础来源配置。
- 实现 Agent 检测占位适配器。
- 实现复制模式同步。
- 完成 `.vsix` 打包。

### 阶段二：可用增强

- 增加 GitHub 来源安装和更新。
- 增加 Windows junction / symlink 同步。
- 增加同步记录和回滚。
- 增加静态质量检测。
- 增加 Webview 管理页面。

### 阶段三：智能分析与 MCP

- 增加语义重叠检测。
- 增加模型分析接口。
- 增加 MCP Server 管理。
- 增加多 Agent 配置模板。
- 增加备份、恢复和迁移能力。

## 10. 风险与约束

- 不同 Agent 工具配置路径和格式变化较快，需要适配器保持可维护。
- Windows 软链接可能受权限限制，需要提供复制模式兜底。
- 普通 VS Code 扩展未必能直接调用 Copilot 当前模型，需要准备外部模型或本地分析替代方案。
- 自动写入其他工具配置存在风险，需要提供预览、确认和回滚机制。
- GitHub 来源结构并不统一，需要做来源解析和容错。

## 11. 验收标准

MVP 完成时应满足：

- 可以在 VS Code 中看到中央仓库 Skills 列表。
- 可以识别每个 Skill 的基本信息。
- 可以检测至少 3 类 Agent 工具或配置目录。
- 可以把一个 Skill 同步到指定目标目录。
- 可以生成并安装 `.vsix`。
- README 中包含安装、使用和打包说明。
