# ASmanager 规划文档

## 1. 背景

不同 AI Agent 工具对 Skills、MCP Server、提示词和工具配置的支持方式不同，导致同一套能力需要在多个工具中重复维护。ASmanager 提供一个 VS Code 内的集中管理入口，以 `~/.agents/skills` 作为统一中央仓库，把 Skills 和 MCP 配置统一安装、更新、检测、同步和分发。

## 2. 目标

- 本机中央 Skills 仓库 `~/.agents/skills`
- 多 GitHub Skills 来源管理与变更检测
- 自动检测本机已安装或已配置的 Agent 工具
- 三种模式同步 Skills 到不同工具目标目录
- Skill 质量检查、Jaccard 语义重叠检测、评分
- MCP Server 配置管理、启停、跨工具同步、健康检查
- 备份恢复、配置导入导出、同步回滚
- 打包为 `.vsix`，方便离线安装和自用分发

## 3. 非目标

- 不做公开 Marketplace 发布
- 不强依赖商业模型 API
- 不自动写入 Agent 配置（MCP 启停除外，有确认）
- 不追求覆盖所有 Agent，优先适配可验证的目录结构

## 4. 核心概念

### 4.1 中央仓库

```text
~/.agents/skills
```

每个 Skill 为独立目录，至少包含 `SKILL.md`。可选：`README.md`、`references/`、`scripts/`、`templates/`。

### 4.2 Skill 来源

内置 4 个来源，支持动态增删：

- `https://github.com/anthropics/skills`
- `https://github.com/openai/skills`
- `https://github.com/MoizIbnYousaf/Ai-Agent-Skills`
- `https://github.com/heilcheng/awesome-agent-skills`

来源本地缓存路径：`~/.agents/skills/.sources/<slug>-<hash>/`（`--depth 1` 浅克隆）。

### 4.3 Agent 适配器

通过检测配置目录是否存在来判断本机是否安装：

| Agent       | 检测路径                       | 技能子目录 |
| ----------- | ------------------------------ | ---------- |
| Claude Code | `~/.claude` `~/.config/claude` | `skills`   |
| Codex       | `~/.codex`                     | `skills`   |
| OpenCode    | `~/.config/opencode`           | `skills`   |
| Cursor      | `~/.cursor` `%APPDATA%/Cursor` | `skills`   |
| Gemini CLI  | `~/.gemini`                    | `skills`   |
| Copilot     | `~/.copilot`                   | `skills`   |
| Cline       | `~/.cline`                     | `skills`   |
| Roo Code    | `~/.roo`                       | `skills`   |

## 5. 已实现功能

### 5.1 Skills 扫描

- 扫描中央仓库子目录（跳过 `.` 开头和 `node_modules`）
- 识别并解析 `SKILL.md`
- 提取 frontmatter：名称、描述、`tags: [...]`、`category:`
- 文件统计：数量、大小、最后修改时间
- 质量评分：名称→30、描述→20、标签→15、分类→10、多文件→20、大小合理→5（满分 100，折算 0–10）
- 标记缺失文档或结构异常的 Skill

### 5.2 来源管理

- 增删改查来源 URL（VS Code 全局配置持久化）
- Git 克隆/拉取（`--ff-only`）
- 来源内 Skill 发现（递归搜索 `SKILL.md`）
- 单个安装 / 批量安装
- 移除来源缓存
- **Git 变更检测**：`fetch` + `rev-list --count HEAD..origin/HEAD`

### 5.3 Agent 检测

- 扫描 10 种工具的常见配置目录
- 展示已检测 / 未检测状态和路径

### 5.4 同步分发

- Copy / Symlink / Junction 三种模式
- 同步记录写入 VS Code 全局状态（最多 200 条）
- 单 Skill 同步 + 批量推送到多 Agent
- **同步回滚**：按记录删除目标目录

### 5.5 质量检测与诊断

- 静态检测：`SKILL.md` 缺失、描述缺失、空目录、文件数 >200
- **Jaccard 词级语义相似度**：≥85% 警告，≥60% 提示；名称相似度 ≥70% 提示

### 5.6 MCP Server 管理

- 扫描 6 种工具的 `mcp.json`（Claude Code / Cursor / Copilot / Cline / Roo Code / Gemini CLI）
- 展示名称、command、状态（enabled/disabled）
- 启停（写回源配置文件的 `disabled` 字段）
- **跨工具同步**：将一个 Server 配置写入另一个工具的 MCP 配置
- **健康检查**：`{command} --version` / `--help` 探活

### 5.7 备份与配置

- 仓库备份：复制到时间戳目录
- 仓库恢复：从备份目录逐个条目恢复
- 配置导出/导入：`centralRepository` + `sources` → JSON

### 5.8 可视化

- Activity Bar 视图容器 `Agent Skills`
- Tree View：`Skills`（可展开子节点查看详情）、`Agents`、`Sources`、`MCP Servers`
- Webview 管理页面：指标仪表盘 + 5 个卡片，绑定了 13 个操作按钮

## 6. 项目结构

```text
asmanager/
  package.json          # 扩展清单，29 条命令
  tsconfig.json
  src/
    extension.ts        # 入口，命令注册 + 管理页面桥接
    types/
      models.ts         # SkillInfo / AgentInfo / SourceInfo / McpServerInfo / SyncRecord 等
    services/
      skillRepository.ts   # Skills 扫描、解析、评分、删除
      sourceManager.ts     # Git 来源管理、变更检测
      agentDetector.ts     # Agent 目录检测
      syncManager.ts       # 同步分发 + 回滚
      mcpManager.ts        # MCP 配置发现、启停、同步、健康检查
      backupService.ts     # 仓库备份/恢复
      diagnostics.ts       # Jaccard 语义重叠检测
      config.ts            # 路径解析、格式化
      fileSystem.ts        # 文件系统工具函数
    views/
      skillsTreeProvider.ts    # Skills 树（展开详情）
      agentsTreeProvider.ts    # Agents 树
      sourcesTreeProvider.ts   # Sources 树
      mcpTreeProvider.ts       # MCP 树
      dashboardWebview.ts      # 管理页面 HTML/CSS/JS 渲染
      treeNode.ts              # 通用树节点
  docs/
    规划文档.md
  README.md
  LICENSE
```

## 10. 风险与约束

- Agent 配置路径和格式可能随工具版本变化，需维护适配器
- Windows 软链接需权限，提供 copy 模式兜底
- GitHub 来源结构不统一，需做来源解析和容错
- 自动写入其他工具配置文件有风险，提供确认和回滚机制
- Git 依赖：来源管理功能需要本机安装 `git`
